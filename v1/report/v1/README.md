### Step 2 — drop the figures into `figures/`

The `.tex` references three figures by filename. Each placeholder in `v1.tex` is
a `\fbox{}` you can search for and replace with `\includegraphics{...}`:

| Filename | Source | How to generate |
|---|---|---|
| `figures/grid_layout.png` | The 10×10 grid visualization | Optional. Either screenshot the README's ASCII grid into a PNG, or render `survivecity_env/layout.py:build_grid()` output via matplotlib (one-line script). |
| `figures/training_curve.png` | Loss/reward across the 12 training steps | **Real plot.** Pull the TensorBoard event file from Hub: `huggingface_hub.hf_hub_download(repo_id='noanya/zombiee', filename='runs/<events.out.tfevents.* file>')` then run `tensorboard --logdir runs/` and screenshot, OR use `tbparse` to load and matplotlib it. |
| `figures/eval_bars.png` | Bar chart: baseline vs trained on the 3 metrics | Auto-generated by eval notebook cell 10 as `eval_step_0012_bars.png`. Download from `noanya/zombiee/eval_results/eval_step_0012_bars.png`. |
| `figures/eval_history.png` | Cross-checkpoint trend (step 7 vs step 12) | Auto-generated by eval notebook cell 12 as `eval_history.png`. Download from `noanya/zombiee/eval_results/eval_history.png`. |

To swap a placeholder for a real figure, replace this:

```latex
\fbox{\parbox[b][3.5cm][c]{0.55\textwidth}{\centering\itshape (placeholder --- training\_curve.png)}}
```

with:

```latex
\includegraphics[width=0.85\textwidth]{figures/training_curve.png}
```

## What the report claims (and what it does NOT claim)

The skeleton is written **honestly**. It does not promise survival-rate gains
that the data may not support. Instead it leads with:

1. **OpenEnv compliance** (the env passed all four R1 validator traps first try)
2. **100% format learning** (0/472 parse failures from the prelim eval)
3. **Emergent ToM broadcast** (the "I notice A2 is very hungry" quote)
4. **Cross-machine training resilience** (HF Hub bridge across Colab/Kaggle/DGX)
5. **Honest limitations section** (reward signal, cue leakage, sample size)

If your final survival numbers ARE strongly positive, edit the abstract and
results to highlight that — change "modest" to "significant" in the conclusion.
If they're flat, the existing wording lands without overclaiming.

## Suggested length

The skeleton compiles to roughly 6 pages. After you fill in TODOs and add the four
figures, expect 7-8 pages. That's about right for a hackathon submission writeup
(judges have to read several of these).

If asked to trim: cut the Behavioural Analysis paragraph to a footnote, and merge
"What Worked" + "Limitations" into one Discussion subsection. That brings it to
~5 pages cleanly.
